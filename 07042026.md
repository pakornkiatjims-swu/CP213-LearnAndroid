### **Part 1: The Core Advanced Views (เนื้อหาเดิมที่นำมาลงรายละเอียด)**

**Mission 1: Compose Animation & Motion**

- **Learn Concept:** * ทำความเข้าใจ `State` ที่ทำให้เกิด Animation ใน Compose
    - การเลือกใช้ API ให้เหมาะสม: `AnimatedVisibility` (เข้า/ออก), `animate*AsState` (เปลี่ยนค่าสี/ขนาด), และ `updateTransition` (เปลี่ยนหลายค่าพร้อมกัน)
- **Vibe Code Prompt Idea:**
    
    "เขียนโค้ด Composable Function สร้างหน้าจอที่มีปุ่ม 'Like' 1 ปุ่ม โดยใช้ Material 3
    **Requirements:**
    
    1. เมื่อกดปุ่ม ขนาดของปุ่มต้องขยายใหญ่ขึ้นเล็กน้อย (Scale animation) แล้วกลับมาขนาดเดิม โดยใช้ `animateFloatAsState` หรือ `spring` animation
    2. สีพื้นหลังของปุ่มต้องเปลี่ยนจาก สีเทา เป็น สีชมพู (ใช้ `animateColorAsState`)
    3. ภายในปุ่มให้ใช้ `AnimatedVisibility` แสดง Icon รูปหัวใจ โผล่ขึ้นมาข้างๆ ข้อความเมื่อปุ่มเปลี่ยนเป็นสถานะ 'Liked'
    **Constraints:** ใช้ State พื้นฐาน (`remember`) ในการเก็บสถานะ ไม่ต้องต่อ ViewModel เขียนให้อยู่ในไฟล์เดียวเพื่อให้ก๊อปปี้ไปรันดูผลลัพธ์ได้ทันที"

**Mission 2: Complex Lists & Pagination**

- **Learn Concept:**
    - การจัดการ `LazyColumn` ขั้นสูง
    - การทำ **Sticky Headers** เพื่อแบ่งกลุ่มข้อมูล (เช่น รายการธุรกรรมแบ่งตามวันที่)
    - **Item Animations:** กลไกการทำให้ Item เลื่อนขึ้น/ลงอย่างนุ่มนวลเมื่อมีการเพิ่มหรือลบข้อมูลใน List
    - **Pagination Concept:** การดักจับ Event เมื่อผู้ใช้เลื่อนหน้าจอมาถึง Item สุดท้าย แล้วสั่งให้ ViewModel โหลดข้อมูลหน้าต่อไป (Load More) พร้อมแสดง Loading Indicator ท้าย List
- **Vibe Code Prompt Idea:**
    
    "เขียนโค้ดจำลองหน้ารายชื่อผู้ติดต่อ (Contact List) ที่รองรับ Sticky Header และ Pagination
    **Requirements:**
    
    1. สร้าง `ViewModel` ที่มี State ถือ List ของ String รายชื่อ (Mock ข้อมูล A-Z) และมี State `isLoading`
    2. สร้าง `LazyColumn` ที่ใช้ `stickyHeader` เพื่อแสดงตัวอักษรนำหน้าชื่อ (A, B, C...)
    3. เมื่อ Scroll ไปถึง Item สุดท้ายของ List ให้ Trigger ฟังก์ชันใน ViewModel เพื่อจำลองการโหลดข้อมูลเพิ่ม (ใช้ `kotlinx.coroutines.delay` หน่วงเวลา 2 วินาที)
    4. ระหว่างโหลด ให้แสดง `CircularProgressIndicator` ที่ด้านล่างสุดของ List
    **Constraints:** ไม่ต้องเชื่อมต่อ API จริง ใช้ Mock Data ใน ViewModel"

**Mission 3: Graphics, Effects & Canvas**

- **Learn Concept:**
    - การใช้งาน `Modifier.graphicsLayer` สำหรับปรับแต่งระดับ Pixel เช่น Alpha, Rotation, Scale
    - **RenderEffect:** การทำภาพเบลอ (Blur) หรือเอฟเฟกต์สีขั้นสูง
    - **Canvas:** การวาดกราฟิกอิสระ (Paths, Shapes, Text) ลงบนหน้าจอโดยตรง ซึ่งมีประโยชน์มากในการทำ Custom UI หรือ กราฟสถิติ
- **Vibe Code Prompt Idea:**
    
    "เขียนโค้ดสร้าง 'กราฟวงกลมแบบโดนัท (Donut Chart)' ด้วย `Canvas` Composable
    **Requirements:**
    
    1. รับค่า Parameter เป็น List ของตัวเลขสัดส่วนเปอร์เซ็นต์ เช่น `listOf(30f, 40f, 30f)` และ List ของสี `Color`
    2. ใช้วิธีการวาด `drawArc` โดยเว้นช่องตรงกลางให้เป็นโดนัท (กำหนด `style = Stroke(...)`)
    3. ทำ Animation ตอนเริ่มต้น ให้กราฟค่อยๆ วาดตัวเองจาก 0 องศา ไปจนถึง 360 องศา (Sweep Angle Animation) เมื่อเปิดหน้าจอขึ้นมา
    **Constraints:** ไม่ใช้ Library วาดกราฟภายนอก ใช้เฉพาะ `androidx.compose.foundation.Canvas` เท่านั้น"

### **Part 2: The Extended Advanced (5 หัวข้อที่เพิ่มเข้ามาใหม่)**

**Mission 4: Advanced Gestures & Interactive UI (การสัมผัสขั้นสูง)**

- **Learn Concept:**
    - ขยับจากการกด (Click) ธรรมดา ไปสู่การลากและการปัดผ่าน `Modifier.pointerInput`
    - การใช้งาน **Swipe-to-Dismiss** (ปัดเพื่อลบ) ใน `LazyColumn` ซึ่งเป็น Standard UX ของแอปมือถือ
    - การทำ UI แบบลากแล้ววาง (Drag and Drop) เบื้องต้น
- **Vibe Code Prompt Idea:**
    
    “เขียนโค้ดสร้างรายการ To-Do List ที่สามารถ 'ปัดเพื่อลบ (Swipe-to-Dismiss)' ได้
    **Requirements:**
    
    1. สร้าง `ViewModel` ที่มี `mutableStateListOf` เก็บรายการข้อความ 5 รายการ
    2. ใช้ `SwipeToDismissBox` (Material 3) สร้าง Item แต่ละบรรทัด
    3. เมื่อผู้ใช้ปัด Item ไปทางซ้าย (End to Start) พื้นหลังของ Item ต้องเปลี่ยนเป็นสีแดง และมี Icon รูปถังขยะ
    4. เมื่อปัดจนสุด ให้ลบข้อมูลนั้นออกจาก State ใน ViewModel ทำให้ UI อัปเดตอัตโนมัติ
    **Constraints:** เขียนให้อยู่ในไฟล์เดียว จัดการ State ให้ถูกต้องตามหลัก MVVM เบื้องต้น"

**Mission 5: Compose Side Effects (หัวใจสำคัญของ Compose + MVVM)**

- **Learn Concept:**
    - เมื่อ UI ไม่ใช่แค่การวาด แต่ต้องเชื่อมโยงกับโลกภายนอก (เช่น การยิง API, การแสดง Snackbar, การหน่วงเวลา)
    - การใช้งาน `LaunchedEffect`: สั่งรัน Coroutine หรือ Trigger Event บางอย่างเมื่อหน้าจอถูกสร้างขึ้น (เช่น สั่ง ViewModel ให้ fetch ข้อมูลเมื่อเปิดหน้า)
    - การใช้งาน `DisposableEffect`: การจัดการทรัพยากรเมื่อ Composable ถูกทำลาย (เชื่อมโยงกับ Lifecycle)
- **Vibe Code Prompt Idea:**
    
    "เขียนโค้ดสาธิตการใช้ `LaunchedEffect` เพื่อแสดง Snackbar เมื่อเกิด Error จาก ViewModel
    **Requirements:**
    
    1. สร้าง `ViewModel` ที่มี `SharedFlow<String>` หรือ `Channel` สำหรับส่งข้อความ Error (One-time event) และมีฟังก์ชันจำลองการเกิด Error
    2. ที่หน้า UI ให้ใช้ `Scaffold` และมี `SnackbarHost`
    3. ใช้ `LaunchedEffect` เพื่อ Observe ค่า Error จาก ViewModel หากมีค่าโผล่มา ให้สั่ง `snackbarHostState.showSnackbar()`
    4. สร้างปุ่ม 'Trigger Error' บนหน้าจอ เพื่อทดสอบการทำงาน
    **Constraints:** เน้นแสดงให้เห็นว่าเราจะไม่ใช้ UI State ธรรมดาในการโชว์ Snackbar แต่จะใช้กลไก Side Effect ของ Compose"

**Mission 6: View Interoperability (ผสานยุคเก่าและยุคใหม่)**

- **Learn Concept:**
    - ในโลกความจริง ไม่ใช่ทุกอย่างจะเขียนด้วย Compose ได้ (เช่น Google Maps, WebView, หรือเครื่องเล่นวิดีโอ ExoPlayer)
    - การใช้งาน `AndroidView` เพื่อนำ Component จาก XML/Legacy Android มาฝังลงในหน้าจอ Jetpack Compose
- **Vibe Code Prompt Idea:**
    
    "เขียนโค้ด Jetpack Compose เพื่อฝัง `WebView` (Android View ดั้งเดิม) ลงในหน้าจอ Compose
    **Requirements:**
    
    1. สร้าง `ViewModel` ที่มี State เก็บค่า URL String (ค่าเริ่มต้นคือ '[https://www.google.com](https://www.google.com/)')
    2. ใช้ Composable `AndroidView` เพื่อสร้าง `android.webkit.WebView`
    3. ตั้งค่า `webViewClient` ให้โหลดเว็บภายในแอป (ไม่เด้งออกไปเบราว์เซอร์ข้างนอก)
    4. สร้าง `TextField` ด้านบนหน้าจอสำหรับพิมพ์ URL และปุ่ม 'Go' เมื่อกดให้ส่ง URL ใหม่ไปอัปเดตที่ ViewModel และให้ WebView โหลดหน้าใหม่
    **Constraints:** โค้ดต้องแสดงวิธีใช้งาน `AndroidView` ร่วมกับ `update` block อย่างถูกต้อง"

**Mission 7: Multi-Activity Transitions (แอนิเมชันข้ามหน้า)**

- **Learn Concept:**
    - เนื่องจาก Tech Stack บังคับใช้ `startActivity` เราจึงต้องจัดการ Animation การเปลี่ยนหน้า (Activity Transitions)
    - การแทนที่แอนิเมชันพื้นฐานของ OS ด้วย Custom Transitions (เช่น Slide-in จากด้านข้าง, Fade-in/Fade-out)
    - การใช้ `ActivityOptionsCompat` เบื้องต้น
- **Vibe Code Prompt Idea:**
    
    “เขียนโค้ดหน้า Activity 2 หน้าด้วย Jetpack Compose (MainActivity และ DetailActivity) เพื่อสาธิตการทำ Activity Transition Animation
    **Requirements:**
    
    1. ที่ `MainActivity` ให้มีปุ่มกดเปิด `DetailActivity` ผ่าน `Intent` โดยมีการส่งค่า String extra ไปด้วย
    2. ใช้คำสั่งจัดการ Transition ของ Android (เช่น `ActivityOptionsCompat.makeCustomAnimation` หรือ `overrideActivityTransition`) เพื่อให้ `DetailActivity` สไลด์เปิดขึ้นมาจากด้านล่างของหน้าจอ (Slide Up)
    3. ที่ `DetailActivity` มีปุ่ม 'Close' เมื่อกดจะเรียกคำสั่ง `finish()` และให้หน้าจอสไลด์กลับลงไปด้านล่าง (Slide Down)
    **Constraints:** ไม่ใช้ Compose Navigation Library (NavHost) แต่ใช้ `startActivity` มาตรฐานตามโจทย์"

**Mission 8: Adaptive Layouts (การรองรับจอหลายขนาด)**

- **Learn Concept:**
    - Mobile ไม่ได้มีแค่มือถือจอแนวตั้งอีกต่อไป (Tablet, Foldable)
    - การใช้ `BoxWithConstraints` เพื่อเช็กขนาดความกว้าง/ความสูงของพื้นที่ปัจจุบัน
    - การปรับ UI อัตโนมัติ: จอเล็กแสดงเป็น `Column`, จอใหญ่แสดงผลแบบ Master-Detail ให้อยู่ใน `Row` เดียวกัน
- **Vibe Code Prompt Idea:**
    
    “

    1. สร้างกล่องสมมติสีเทาแทน 'รูปโปรไฟล์' และ Text อธิบาย 'ข้อมูลส่วนตัว'
    2. หากความกว้างหน้าจอ (maxWidth) **น้อยกว่า 600.dp** (แนวตั้ง/มือถือ): ให้แสดงผลแบบ `Column` (รูปอยู่บน ข้อมูลอยู่ล่าง)
    3. หากความกว้างหน้าจอ (maxWidth) **มากกว่า 600.dp** (แนวนอน/แท็บเล็ต): ให้แสดงผลแบบ `Row` (รูปอยู่ซ้าย ข้อมูลอยู่ขวา จัดพื้นที่แบบน้ำหนัก weight)
    **Constraints:** โค้ดต้องรันและทดสอบการหมุนหน้าจอ (Orientation change) เพื่อดูผลลัพธ์การเปลี่ยน Layout ได้ทันที"
